Part 03 - A03

PWD Functional requirements

I implemented my application following the single page application approach which is an app that does not need to reload the page during its use and works within a browser

To fulfill the functionality of multiple windows within the application, I implemented a class named draggableElement.js that can create a draggable window and able to take other classes in it. I got the idea of my implementation from the W3Schools website.

So basically the window created by this class is draggable and movable inside the PWD. Moreover, I have an icon in the navbar for each application, each icon opens its application by clicking on it, and for each application there is a close button icon on the right upper corner of the window. This close button icon uses a method called removeHandler () to close the desirable window.

As for the focus functionality, I implemented two methods inside draggableElement.js named focusWindowHandler () and unFocusWindowHandler (), those two methods add and remove a red solid border to the perimeter of the app window.

PWD Non functional requirements

In my git commit history, I have 44 commits, so basically, each time I brought some changes to my code or I add new class I commit my changes and push it to git lab. Hence it is easy to roll back to my old code in case something went wrong

My code is following standard.js code standard. For example I am using single quotes for strings, use two spaces for indentation etc.

I used JSDoc to comments all my functions and class. My linters is passing without errors when i run npm test, the code is following ES modules approach and the application is visually appealing by using css.

Memory game window application

For the memory game i created a Memory.js file which contains an exported function, and inside this function i have all the needed method to create the game that can be adjustable to any desirable size. For instance I implemented a method called turnBrickHandler () to turn the bricks and remove them in case they are similar or turn them back in case they are not. Along with the shuffleArray () method which creates a shuffles picture array for the game.

The user is able to open and play multiple memory games simultaneously by the helping of draggableElement class. Basically each time the user click on the memory game icon the draggableElement class created a new window and new instance of the game inside this window. The user can play the game using the mouse and each game has counter to count the number of tries that user did to win the game

Chat window application

For the chat application, I created a chat-app.js class which inherited the window.HTMLElement class. Inside this class, I created the proper constructor and used the information provided by the teacher like the web socket and API to connect with the server.
Also, I created all the needed method to make this app work properly such as the receiveHandler () method which handles the received data form websocket by formatting it and then show it to the user also specify the maximum number of messages, messageTimeHandler () which shows the time of chat next to messages and many other methods.

The user is able to have several chat applications running at the same time by the helping of draggableElement class.
When the user open the application for the first time, user is asked to enter his/her username with the helping of sendHandler () method which checks user name and save it in the local storage. So the username remains the same the next time the user starts a chat application or the PWD is restarted.
The user is be able to send chat messages using a textarea, The user is also able to see up to 50 latest messages since the chat applications was opened. Also the user is able to change the user name in any time.
